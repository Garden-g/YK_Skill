# Alibaba Inquiry Record

本技能用于把指定时间段的国际站询盘整理进月度 Excel 台账。它只做记录、筛选、去重、追加和月度表格更新，不做询盘复盘会、不做店铺经营分析，也不执行任何会影响客户或店铺状态的操作。

## 核心原则

- 只记录真实获取到的询盘和会话信息；客户、需求、国家、等级、联系方式、业务员和下单情况都不能编造。
- 用户说"上周"、"本周一"、"昨天"、"最近 N 天"或明确日期时，按该时间范围采集并记录。
- 每个月一个 Excel 文件，文件名建议为 `询盘记录_YYYY-MM.xlsx`。
- 同一个客户在同一个月份只保留一行；后续同月新消息合并进原行，不重复新增。
- 低质客户、广告推广、群发营销、封禁客户、黑名单客户、明显非采购意向联系人不进入主表。
- 表格可见字段必须按用户模板保留；辅助字段可以隐藏，用于去重和连续追加。
- 最终回复用户时只说业务结果和文件位置，不展示采集过程、技术细节或原始报错。

## 可见字段

Excel 主表必须按下面顺序展示字段：

| 字段 | 填写要求 |
|---|---|
| 日期 | 询盘或关键消息日期。跨多天合并时保留多个日期，按时间顺序用 `；` 分隔。 |
| 图片 | 询盘关联商品图、会话卡片图、用户提供的图片链接或本地图片路径；没有就留空。 |
| 客户 | 买家名、联系人名或可识别客户名称。 |
| 业务 | 接待业务员、负责人或账号名称；同月多业务员接待同一客户时合并。 |
| 国家 | 客户国家或地区；无法确认就留空，不猜测。 |
| 需求 | 客户真实表达的采购需求、订单问题、规格、数量、样品、目标价、交付要求等。 |
| 等级 | 客户等级，例如 L1、L2、L3；无法确认就留空。 |
| 下单情况 | 已下单、待报价、待样品、待跟进、复购、未下单等真实状态。 |
| 公司名称、网站和个人信息 | 公司名、网站、联系人身份、邮箱、电话、社媒等可确认信息。 |
| 是否添加 WhatsApp，电话沟通等 | WhatsApp、电话、邮箱、线下沟通、未添加等联系状态。 |
| 客户类型 | 标品/常规品客户、定制客户、复购客户、样品客户、潜在客户等。 |

隐藏辅助字段：

- 去重键
- 会话/询盘来源ID
- 记录月份
- 首次记录时间
- 最后更新时间
- 筛选原因

## 时间范围规则

| 用户说法 | 解释 |
|---|---|
| 今天 | 今天 00:00 到当前时间 |
| 昨天 | 昨天全天 |
| 本周、本周至今 | 本周一 00:00 到当前时间 |
| 本周一、本周二等 | 本周对应星期的全天 |
| 上周 | 上一个完整自然周，周一 00:00 到周日 23:59 |
| 最近 N 天、近 N 天 | 从当前时间向前滚动 N 天 |
| 4 月、2026 年 4 月 | 该自然月整月 |
| 明确日期区间 | 严格按用户给出的开始和结束日期 |

如果时间范围跨月，必须拆分写入对应月份的 Excel 文件。

## 工作流程

### 1. 明确记录范围

从用户原话提取时间范围、是否有指定业务员、是否只看某类客户、是否要排除特定客户。

如果时间范围可以按上表直接确定，不要追问；直接执行并在最终回复里说明记录周期。

### 2. 获取询盘和会话事实

只读取与询盘记录有关的信息：

- 询盘或会话发生时间。
- 客户名称、国家、等级、公司和网站。
- 接待业务员或负责人。
- 客户需求、采购意向、规格、数量、样品、报价、订单或复购信息。
- 联系方式与是否添加 WhatsApp、电话、邮箱等。
- 关联图片或商品卡片图片。

如果当前环境无法读取真实询盘或会话记录，停止生成正式记录，改为输出"本次无法获取真实询盘数据，需要补充数据来源"。不要用示例客户填表。

### 3. 过滤不应记录的客户

命中以下任一情况时，不写入主表：

- 明显广告推广：卖 SEO、建站、推广服务、展会招商、贷款、投资、加密货币、群发营销等。
- 低质无采购意向：只有普通寒暄、没有产品需求、无法判断采购意图，且后续没有追问。
- 已封禁或黑名单：记录里明确出现封禁、拉黑、黑名单、风险客户等状态。
- 非买家身份：服务商、同行推销、招聘、无关咨询。

筛选要保守：只在证据明确时排除。模糊但可能是真实采购的客户应保留，并在需求或客户类型里标注"待确认"。

### 4. 标准化记录

把每条有效询盘整理成一条记录，至少包含日期、客户和需求。其他字段按真实信息补充：图片、业务、国家、等级、下单情况、公司网站与个人信息、WhatsApp/电话沟通情况、客户类型。

字段缺失时留空，不要猜。可以保留稳定来源标识用于辅助追踪，但不要展示在主表可见区域。

### 5. 更新月度 Excel

用随附程序更新表格：

```bash
python3 scripts/update_inquiry_monthly_xlsx.py --input records.json --output-dir .
```

记录文件里至少准备日期、客户、需求三类信息；其他字段有真实信息就补充，没有就留空。

程序会自动：

- 按日期拆分月份。
- 创建或读取 `询盘记录_YYYY-MM.xlsx`。
- 按"客户 + 月份"去重。
- 把同月重复客户的新信息合并到原行。
- 隐藏辅助列。
- 给表头、重复合并记录和待确认信息添加样式。
- 保存后校验 Excel 文件能被正常读取。

### 6. 交付前检查

交付 `.xlsx` 前必须完成：

- 文件能被解压校验。
- 文件能被 Excel 读取库打开。
- 包内没有残留的表格对象、空 drawing 或异常关系。
- 如果执行环境提供 LibreOffice，先用 LibreOffice headless 重存，再做上述检查。

任一步失败都不要交付，先修复并重新生成。

## 输出格式

最终回复用户时保持简洁：

```markdown
已更新询盘记录：

- 周期：YYYY-MM-DD 至 YYYY-MM-DD
- 月表：询盘记录_YYYY-MM.xlsx
- 新增：N 条
- 合并重复客户：N 条
- 筛除低质/广告/封禁客户：N 条

已筛除内容没有进入主表；同月同客户已合并到原记录。
```

如果缺少真实数据：

```markdown
这次还不能生成正式询盘记录，因为当前没有拿到真实询盘/会话数据。请提供导出的询盘记录、会话列表，或授权后再重新执行。不会用示例客户填表。
```

## 禁止事项

- 不要发送客户消息。
- 不要修改客户资料。
- 不要发布或编辑商品。
- 不要修改店铺、账号、自动接待、跟进或营销配置。
- 不要把广告、订单、店铺经营分析扩展进本技能。
- 不要在最终结果里写采集过程、技术细节或技术报错。
